# DevFlow 작업 진행 순서

## 문서 목적

이 문서는 **DevFlow = 개발자용 집중시간 기록 + 룰기반 분석 서비스**를
포트폴리오 수준의 MVP로 구현하기 위한 작업 순서를 정리한 문서다.

핵심 방향은 단순 타이머 앱이 아니라,
**기록된 집중 데이터를 분석해 인사이트를 제공하는 서비스**로 완성하는 것이다.

---

## 우선순위 원칙

1. 먼저 서비스의 핵심 흐름이 동작해야 한다.
2. 그다음 데이터가 쌓이도록 기록 구조를 완성한다.
3. 이후 집계와 분석을 붙여 DevFlow의 차별점을 만든다.
4. 마지막에 UI 완성도, 반응형, 배포 품질을 끌어올린다.

---

## 최종 MVP 목표

아래 기능이 연결되면 1차 포트폴리오 MVP로 본다.

- 회원가입 / 로그인
- 집중 세션 시작 / 종료
- 세션 기록 저장
- 오늘 대시보드
- 주간 집계
- 기본 인사이트 3종
  - 집중 시간 수준 분석
  - 시간대 분석
  - 루틴 분석

---

## 작업 순서

### 1. 프로젝트 기반 정리

목표:
개발 전반에서 흔들리지 않도록 서비스 구조와 구현 범위를 먼저 고정한다.

작업:
- 기술 스택 확정
  - `Next.js`
  - `TypeScript`
  - `Tailwind`
  - `Supabase`
- 공통 폴더 구조 정리
- 환경 변수 목록 정리
- 기본 라우트 구조 정의
- 공통 타입 초안 작성

산출물:
- 프로젝트 기본 구조
- 환경 변수 예시
- 페이지/기능 단위 폴더 기준

---

### 2. 인증 기능 구현

목표:
사용자별 기록과 대시보드를 만들기 위한 로그인 기반을 완성한다.

작업:
- 회원가입
- 로그인
- 로그아웃
- 인증 상태 유지
- 보호 라우트 처리

우선 화면:
- 랜딩 페이지
- 로그인 페이지
- 회원가입 페이지

산출물:
- 인증 가능한 기본 사용자 흐름
- 로그인 후 대시보드 진입 구조

---

### 3. DB 스키마 설계 및 연결

목표:
핵심 데이터를 안정적으로 저장할 수 있는 구조를 만든다.

우선 테이블:
- `users`
- `categories`
- `sessions`
- `daily_goals`

후순위 테이블:
- `weekly_reports`
- `streaks`

작업:
- 테이블 생성
- 기본 카테고리 seed 구성
- 세션 상태/난이도 enum 또는 제약조건 정의
- DB 타입과 프론트 타입 연결

산출물:
- MVP 기준 DB 구조
- 세션 기록 저장 가능 상태

---

### 4. 타이머 핵심 흐름 구현

목표:
DevFlow의 핵심 입력 수단인 집중 세션 흐름을 먼저 완성한다.

작업:
- 집중 세션 시작
- 진행 중 타이머 표시
- 일시정지
- 종료
- 취소
- 기본 프리셋 적용
  - 25분 집중 / 5분 휴식
- 사용자 커스텀 시간 입력

중요 포인트:
- 세션 종료 후 반드시 기록 입력 단계로 이어지게 설계한다.
- 단순 종료가 아니라 데이터 저장 흐름의 시작점으로 본다.

산출물:
- 작동하는 집중 세션 화면
- 세션 상태 변경 API

---

### 5. 세션 기록 입력 기능 구현

목표:
분석 가능한 데이터를 쌓기 위해 세션 메타데이터 입력을 완성한다.

입력 항목:
- 작업명
- 작업 카테고리
- 난이도
- 집중도 자가평가
- 메모

작업:
- 세션 종료 후 기록 입력 모달 또는 폼 노출
- 기록 저장 API 연결
- validation 처리
- 기록 누락 방지 UX 적용

산출물:
- 의미 있는 세션 데이터 저장 구조
- 이후 분석에 사용할 기반 데이터 확보

---

### 6. 세션 히스토리 화면 구현

목표:
사용자가 저장한 기록을 확인하고 관리할 수 있게 한다.

작업:
- 오늘 세션 목록
- 날짜별 히스토리 리스트
- 카테고리 필터
- 완료/취소/중단 상태 필터
- 메모 확인

산출물:
- 기록 조회 가능한 히스토리 페이지
- 대시보드 외 기본 데이터 확인 수단

---

### 7. 일간 대시보드 구현

목표:
사용자가 오늘의 집중 상태를 바로 이해할 수 있게 한다.

표시 항목:
- 오늘 총 집중 시간
- 완료 세션 수
- 가장 많이 한 카테고리
- 최고 집중 시간대
- 오늘의 한 줄 인사이트

작업:
- 오늘 기준 집계 로직 작성
- 대시보드 카드 UI 구현
- 어제 대비 변화율 계산

산출물:
- 로그인 후 바로 가치가 보이는 메인 화면

---

### 8. 주간 집계 로직 구현

목표:
DevFlow의 핵심 가치인 "패턴 확인"을 보여주는 리포트 기반을 만든다.

집계 항목:
- 주간 총 집중 시간
- 요일별 집중 시간
- 시간대별 세션 수
- 시간대별 평균 집중도
- 카테고리별 총 시간
- 평균 세션 길이
- 목표 달성률

작업:
- 주간 범위 계산
- 집계 API 또는 서버 함수 작성
- 대시보드/리포트용 응답 타입 정리

산출물:
- 주간 리포트 데이터 구조
- 차트 연결 가능한 가공 데이터

---

### 9. 룰 기반 분석 로직 구현

목표:
DevFlow를 단순 기록 앱이 아니라 분석 서비스로 보이게 만드는 핵심 기능을 완성한다.

MVP 우선 분석 규칙:
- 집중 시간 수준 분석
- 시간대 생산성 분석
- 루틴 안정성 분석

2차 분석 규칙:
- 카테고리 편중 분석
- 세션 유지력 분석
- 회복/슬럼프 감지
- 자기평가 일치도 분석

작업:
- 규칙별 계산 함수 분리
- 분석 결과 타입 정의
- 템플릿 기반 문장 생성기 작성
- 다중 인사이트 우선순위 정리

추천 구조:
- `lib/analytics/metrics`
- `lib/analytics/rules`
- `lib/analytics/templates`
- `lib/analytics/generator`

산출물:
- 주간 인사이트 카드
- 오늘의 한 줄 분석
- 재사용 가능한 분석 엔진 구조

---

### 10. 리포트 화면 구현

목표:
누적 데이터와 분석 결과를 시각적으로 설득력 있게 전달한다.

필수 위젯:
- 주간 총 집중 시간 카드
- 요일별 막대 그래프
- 시간대 히트맵 또는 대체 차트
- 카테고리 도넛 차트
- 인사이트 카드 3~5개
- 지난주 대비 변화율

작업:
- 차트 라이브러리 연결
- 리포트 레이아웃 구현
- 반응형 대응
- 로딩/빈 상태/데이터 없음 상태 처리

산출물:
- 포트폴리오에서 보여주기 좋은 핵심 화면

---

### 11. 목표 및 스트릭 기능 구현

목표:
사용자가 기준점과 성취 흐름을 볼 수 있게 만든다.

작업:
- 일일 집중 시간 목표 설정
- 주간 기록일 목표 설정
- 목표 달성률 계산
- 연속 기록일 계산

산출물:
- 동기부여 요소 강화
- 대시보드 체류 가치 상승

---

### 12. 마무리 품질 작업

목표:
실제 서비스처럼 보이도록 사용성과 완성도를 정리한다.

작업:
- 반응형 UI 점검
- 빈 상태 메시지 정리
- 에러 처리 정리
- 로딩 UX 정리
- 색상/타이포그래피 정리
- 메타데이터 및 랜딩 문구 정리
- README 작성 또는 보강
- 배포

산출물:
- 포트폴리오 제출 가능한 수준의 완성도

## 구현 시 주의할 점

- 처음부터 기능을 많이 넣지 말고, "기록 → 집계 → 분석 → 시각화" 흐름을 먼저 완성한다.
- 분석 기능은 AI 의존 없이도 충분히 설득력 있게 만들 수 있으므로 규칙 기반 품질에 집중한다.
- 차트는 예쁜 것보다 해석 가능한 구조가 중요하다.
- 포트폴리오에서는 기능 수보다 문제 정의와 데이터 활용 방식이 더 강하게 보인다.
