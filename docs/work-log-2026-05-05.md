# DevFlow 작업 로그

작성일: 2026-05-05

## 작업 범위

이번 작업은 이전 회차에서 확정한 카테고리 비활성화 정책과 Settings preset 설계를 실제 구현으로 연결했다.

## 시작 브랜치

- `feat/settings-presets-category-active`

## 작업 내역

- `docs/sql/002_settings_presets_and_category_active.sql` migration 문서를 추가했다.
- `docs/sql/001_mvp_foundation.sql`에도 신규 설치 기준 스키마를 반영했다.
- `categories.is_active` 타입을 Supabase 타입과 도메인 타입에 반영했다.
- `user_settings` 타입과 `UserSettings` 도메인 타입을 추가했다.
- Settings page data query에서 `user_settings`를 조회하도록 확장했다.
- Timer page data query에서 `user_settings`를 조회하고, 새 세션 카테고리 선택지는 active 카테고리만 전달하도록 변경했다.
- `upsertUserSettingsAction`을 추가해 기본 집중/휴식 시간을 저장할 수 있게 했다.
- `deactivateCategoryAction`을 추가해 사용자 카테고리를 실제 삭제 대신 비활성화할 수 있게 했다.
- `/settings`에 기본 집중/휴식 시간 입력 UI와 카테고리 비활성화 UI를 추가했다.
- `/timer` 시작 폼의 초기 집중/휴식 시간을 저장된 preset 기준으로 표시하게 했다.
- 세션 완료 액션에서 비활성 또는 타 사용자 카테고리가 새 완료 기록에 연결되지 않도록 검증을 추가했다.

## 검증

- `npm run lint` 통과
- `npm run build` 통과
- `npm run dev` 로컬 서버 기동 확인
- `curl http://localhost:3000` 200 응답과 홈 콘텐츠 확인
- `curl http://localhost:3000/auth/login` 200 응답과 로그인 폼 콘텐츠 확인
- `curl http://localhost:3000/settings` 비로그인 상태에서 `/auth/login` 307 리다이렉트 확인
- `curl http://localhost:3000/timer` 비로그인 상태에서 `/auth/login` 307 리다이렉트 확인
- `agent-browser open http://localhost:3000`은 현재 환경에서 daemon startup 실패로 실행하지 못했다.
