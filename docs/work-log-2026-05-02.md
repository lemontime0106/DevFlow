# DevFlow 작업 로그

작성일: 2026-05-02

## 작업 범위

이번 작업은 1회차 계획 수립 범위를 실제 문서로 고정하는 작업이다.
구현에 들어가기 전에 카테고리 삭제/비활성화 정책과 Settings 기본 집중/휴식 시간 저장 구조를 확정했다.

## 확인한 코드 기준

- `docs/sql/001_mvp_foundation.sql` 기준 `categories`에는 `is_active`가 없다.
- `sessions.category_id`는 `categories.id`를 참조하고, 카테고리 삭제 시 `on delete set null`이다.
- RLS에는 사용자 카테고리 delete 정책이 있지만, 현재 UI와 서버 액션에는 삭제 기능이 없다.
- `app/settings/actions.ts`에는 카테고리 생성/수정 액션과 일일 목표 저장 액션만 있다.
- `app/settings/page.tsx`는 기본/사용자 카테고리 표시와 사용자 카테고리 수정 UI를 제공한다.
- `lib/types/database.ts`에는 `user_settings` 타입이 아직 없다.

## 결정한 정책

### 카테고리

- 실제 삭제보다 `is_active` 기반 비활성화를 우선한다.
- 기본 카테고리는 비활성화할 수 없다.
- 사용자 카테고리만 비활성화할 수 있다.
- 비활성 카테고리는 새 세션 선택 목록에서 제외한다.
- 비활성 카테고리가 연결된 기존 세션은 히스토리와 리포트에서 계속 표시한다.

### Settings preset

- 기본 집중 시간과 기본 휴식 시간은 `user_settings` 테이블에 저장한다.
- `daily_goals`는 날짜별 목표 테이블로 유지한다.
- `users` 테이블은 인증 프로필 성격을 유지하고 설정 필드 확장은 피한다.
- 기본값은 집중 25분, 휴식 5분으로 둔다.
- validation 기준은 집중 1~180분, 휴식 0~60분으로 둔다.
