### 4. 일간 대시보드 집계 로직 구현

- 오늘 총 집중 시간

- 가장 많이 한 카테고리
- 최고 집중 시간대
- 최근 세션 목록

관련 파일:

- `app/dashboard/page.tsx`
- `lib/sessions/queries.ts`

### 5. Next.js 16 blocking route 대응

- 인증/세션 조회를 페이지 최상단에서 직접 await 하던 구조 정리
- `Suspense` 경계 안쪽의 async content 방식으로 변경
- `/dashboard`, `/timer`, `/history`, `/auth/login`, `/auth/sign-up` 적용

### 6. 헤더 브랜드 링크 동작 정리

- 로그인 상태에서는 메인 브랜드 영역 클릭 시 `/dashboard`
- 비로그인 상태에서는 `/`
- 앱 셸과 랜딩 페이지에서 동일한 동작 사용

관련 파일:

- `components/layout/brand-link.tsx`
- `components/layout/app-shell.tsx`
- `app/page.tsx`

### 7. 타이머 프리셋 버튼 오류 수정

- `formAction`이 있는 버튼에 `name`을 함께 지정해서 발생한 React 경고 해결
- 프리셋 버튼을 로컬 state 변경용 버튼으로 수정
- 실제 서버 액션은 기본 제출 버튼만 사용하도록 정리

### 8. 탭 이동 성능 원인 점검 및 일부 경량화

- 사용자별 페이지가 모두 동적 데이터에 의존하고 있어 정적 캐시 적용이 어려운 점 확인
- 불필요한 인증 조회 중복 일부 제거
- `getAuthUser()`를 `getClaims()` 중심으로 단순화
- 앱 셸의 브랜드 링크는 인증 앱 구간에서 불필요한 인증 조회 없이 `/dashboard`로 직접 연결
